# `test_cases.md`

本文件提供 `cryskill-title-matching` 的示例输入。

目标不是给标准答案，而是说明：

- 这套 Skill 可以处理什么输入
- 不同使用模式该怎么喂数据
- 输出应该关注哪些字段

---

## Case 1. `topic_to_title`

### 输入

话题：

`我命由我不由天，出生在海边，是命；但大海是天堑还是通途，你定。`

### 预期任务

- 先把抽象话题压缩成清晰命题
- 判断主类和主方法
- 匹配模板
- 输出 `<=21字` 标题

### 预期关注点

- 是否判成 `解释重构 > 困境归因重构型`
- 是否能避免把它直接写成口号
- 是否能匹配到合适模板而不是自由乱写

## Case 2. `content_to_title`

### 输入

正文内容：

`很多酒店都想做疗愈，但真正能被用户记住、愿意买单的，往往不是多加几节疗愈课，或多请一位疗愈老师。

更值得被重新设计的，其实是酒店里那些长期被忽视的高价值时段。比如“睡前两小时”。

白天的水疗、健康餐饮、瑜伽课程已经越来越拥挤，夜晚反而提供了一个新的产品窗口：低刺激、高记忆点，也更贴近用户真正的恢复需求。

围绕睡前这段时间，酒店可以重新组织出一条完整的产品线，比如音乐疗愈、香气、热疗、水疗、呼吸练习、沐浴仪式、助眠茶饮，甚至与品牌共创更完整的夜间恢复方案。

这背后的关键，不是项目变多了，而是酒店开始把睡前状态当成一个可以被设计、被定价、被售卖的场景。

对酒店来说，是新的消费入口
对疗愈服务方来说，是新的合作方案
对品牌来说，这也是通过产品可嵌入的新场景。

未来酒店疗愈服务真正拉开差距的地方，或许不在白天，而在夜晚。`

### 预期任务

- 提炼核心承诺
- 判断主类和主方法
- 匹配最适合的模板
- 输出多个商业表达标题

### 预期关注点

- 是否判成 `路径捷径 > 新路径发现型`
- 是否能保留“睡前两小时/夜晚”这个具体场景
- 是否避免写成行业报告句

## Case 3. `optimize_existing_title`

### 输入

原标题：

`被忽视的睡前仪式感，正在成为新的消费场景`

### 预期任务

- 判断原标题当前命中的方法
- 识别弱点
- 保留核心意思并优化成更像标题的话术

### 预期关注点

- 是否指出原标题偏公文感
- 是否能把“新的消费场景”压缩成更锋利表达
- 是否确保最终标题 `<=21字`

## Case 4. `analyze_reference_title`

### 输入

高数据标题：

`有些痛苦，是你选的`

### 预期任务

- 归类
- 解释 why it works
- 说明边界
- 判断它适合沉淀为哪种模板

### 预期关注点

- 是否判成 `认知颠覆 > 因果质疑型`
- 是否说明它为什么不是单纯的情绪安抚
- 是否能沉淀到 `有些X，是你Y的` 母版

## Case 5. `analyze_reference_title`

### 输入

高数据标题：

`妈妈没教，但女孩得知道的小知识`

### 预期任务

- 判断主类和辅类
- 说明为什么它更像“信息壁垒”，而不是“身份归属”

## 建议使用方式

在调试 Skill 时，建议每次测试都检查以下几点：

1. 主类是否稳定
2. 主方法是否稳定
3. 是否误把辅类当主类
4. 是否匹配到了合理模板
5. 最终标题是否 `<=21字`
6. 标题是否像标题，而不是像内容结论
