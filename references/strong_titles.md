# `strong_titles.md`

本文件用于收录高质量强样本，作用不是堆积标题，而是校准：

- 什么样的标题真的有强互动表现
- 它命中了哪一类主驱动力
- 它为什么有效
- 它和相邻方法的边界在哪里

---

## 当前观察

首批强样本已经暴露出两个对后续很重要的事实：

1. `output_schema.md` 里需要严格区分：
   - `primary_category`
   - `primary_method`
   - `secondary_category`
   - `secondary_role`

2. reference 标注过程中会自然暴露标签缺口和命名不稳：
   - 例如“查没查出什么，对你都没好处。”本质上仍应落回 `认知颠覆 > 因果质疑型`
   - 说明 reference 文件不是只做展示，也承担“反向修正规则和方法卡”的作用
